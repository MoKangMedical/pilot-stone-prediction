# 飞行员结石风险评估系统 - 数据导入方案

## 概述

本文档介绍如何将外部采集的评估数据导入到飞行员结石风险评估系统。系统支持三种数据导入方式：**CSV/Excel文件导入**、**REST API接口**和**第三方系统集成**。

---

## 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **CSV/Excel导入** | 操作简单、无需编程、支持批量导入 | 需要手动上传、容易出错 | 小规模数据迁移、一次性导入 |
| **REST API接口** | 自动化、实时同步、易于集成 | 需要编程、需要认证管理 | 定期数据同步、第三方系统集成 |
| **Webhook推送** | 实时性强、自动触发 | 需要外部系统支持、网络依赖 | 实时数据流、事件驱动 |

## 方案一：CSV/Excel文件导入

### 1.1 数据格式要求

#### CSV格式示例
```csv
pilotName,pilotCode,gender,age,flightDuration,annualFlightHours,serumOxalate,uricAcid,calcium,phosphorus,urinePH,urineGravity,creatinine,urea,totalCholesterol,hdl,ldl,triglycerides,height,weight,waist,hip,bodyFat,systolic,diastolic
张三,CA123456,male,35,10,800,3.5,380,2.3,1.2,6.5,1.020,90,5.5,4.2,1.3,2.5,1.8,175,75,85,95,18,120,80
李四,CA123457,female,28,5,600,2.8,350,2.1,1.1,6.8,1.018,85,5.2,3.9,1.5,2.2,1.5,165,62,78,88,20,115,75
```

#### Excel格式要求
- 第一行为列标题
- 支持的列名称（必填项用*标记）：
  - 个人信息：`pilotName*`、`pilotCode*`、`gender*`、`age*`
  - 职业特征：`flightDuration*`、`annualFlightHours*`
  - 代谢指标：`serumOxalate`、`uricAcid`、`calcium`、`phosphorus`、`urinePH`、`urineGravity`、`creatinine`、`urea`、`totalCholesterol`、`hdl`、`ldl`、`triglycerides`
  - 健康指标：`height`、`weight`、`waist`、`hip`、`bodyFat`、`systolic`、`diastolic`

### 1.2 导入步骤

1. **准备数据文件**
   - 将数据整理为CSV或Excel格式
   - 验证所有必填项都有值
   - 检查数据类型和范围

2. **上传文件**
   - 登录系统后进入"数据导入"页面
   - 选择"CSV/Excel导入"选项
   - 点击"选择文件"上传数据文件

3. **数据验证**
   - 系统自动验证数据格式和内容
   - 显示验证结果和错误提示
   - 允许用户修正后重新上传

4. **确认导入**
   - 预览导入数据
   - 确认无误后点击"导入"按钮
   - 系统将数据保存到数据库

### 2.3 集成示例

#### Python集成示例
```python
import requests
import json

class PilotAssessmentClient:
    def __init__(self, base_url, api_key):
        self.base_url = base_url
        self.headers = {
            "X-API-Key": api_key,
            "Content-Type": "application/json"
        }
    
    def create_assessment(self, assessment_data):
        """创建单个评估记录"""
        url = f"{self.base_url}/api/trpc/assessment.create"
        response = requests.post(url, json=assessment_data, headers=self.headers)
        return response.json()
    
    def batch_import(self, assessments):
        """批量导入评估记录"""
        url = f"{self.base_url}/api/trpc/assessment.importBatch"
        payload = {"assessments": assessments}
        response = requests.post(url, json=payload, headers=self.headers)
        return response.json()
    
    def list_assessments(self, skip=0, take=10):
        """查询评估记录列表"""
        url = f"{self.base_url}/api/trpc/assessment.list"
        params = {"skip": skip, "take": take}
        response = requests.get(url, params=params, headers=self.headers)
        return response.json()

# 使用示例
client = PilotAssessmentClient(
    base_url="https://your-domain.com",
    api_key="your_api_key"
)

# 创建单个评估
assessment = {
    "pilotName": "张三",
    "pilotCode": "CA123456",
    "gender": "male",
    "age": 35,
    "flightDuration": 10,
    "annualFlightHours": 800,
    "metabolicData": { /* ... */ },
    "healthData": { /* ... */ }
}
result = client.create_assessment(assessment)
print(result)
```

## 方案三：第三方系统集成

### 3.1 Webhook推送

如果您的系统需要实时推送评估数据，可以配置Webhook回调：

#### 配置Webhook
```
POST /api/admin/webhooks/register
Content-Type: application/json
Authorization: Bearer {admin_token}

{
  "url": "https://your-external-system.com/webhooks/assessment",
  "events": ["assessment.created", "assessment.updated"],
  "secret": "your_webhook_secret"
}
```

#### Webhook事件格式
```json
{
  "event": "assessment.created",
  "timestamp": "2026-01-21T10:30:00Z",
  "data": {
    "id": 123,
    "pilotName": "张三",
    "pilotCode": "CA123456",
    "riskScore": 23.5,
    "riskLevel": "medium",
    "assessmentDate": "2026-01-21T10:30:00Z"
  },
  "signature": "sha256=..."
}
```

#### 验证Webhook签名
```python
import hmac
import hashlib
import json

def verify_webhook(payload, signature, secret):
    """验证Webhook签名"""
    expected_signature = hmac.new(
        secret.encode(),
        payload.encode(),
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(signature, expected_signature)

# 使用示例
payload = request.get_data(as_text=True)
signature = request.headers.get('X-Webhook-Signature')
if verify_webhook(payload, signature, 'your_webhook_secret'):
    data = json.loads(payload)
    # 处理数据
else:
    # 签名验证失败
    pass
```

### 3.2 定时同步

对于需要定期同步数据的场景，可以使用定时任务：

## 常见问题

**Q: 导入的数据如何处理重复项？**
A: 系统会根据`pilotCode`和`assessmentDate`判断重复，提供覆盖或跳过的选项。

**Q: 导入失败后如何处理？**
A: 系统会生成详细的错误报告，显示失败的行号和原因，支持修正后重新导入。

**Q: 导入的数据是否会影响现有数据？**
A: 不会。导入是追加操作，不会修改或删除现有数据。

**Q: 如何确保数据安全？**
A: 所有API调用都需要认证，支持HTTPS加密传输，所有操作都有审计日志。
