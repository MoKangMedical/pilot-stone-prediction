# 飞行员肾结石风险评估系统 - 已知问题和改进方向

## 已知问题

### 1. PDF导出兼容性问题

**问题描述**：html2canvas库不支持OKLCH颜色格式，导致PDF导出时颜色显示不正确。

**影响范围**：
- PDF报告生成功能
- 打印功能可能受到影响
- 某些复杂CSS效果无法正确渲染

**当前解决方案**：
- 已将所有OKLCH颜色替换为RGB/HEX格式
- 在 `client/src/index.css` 中使用标准颜色定义
- 颜色映射已在版本 `ad5ec58b` 中完成

**后续改进建议**：
- 考虑使用其他PDF生成库（如Puppeteer、WeasyPrint）
- 实现服务端PDF生成，避免浏览器兼容性问题
- 添加PDF导出预览功能，让用户在导出前检查效果

**相关文件**：
- `client/src/index.css` - 全局样式定义
- `client/src/components/PrintReport.tsx` - PDF导出组件
- `server/report-generator.ts` - 报告生成逻辑

---

### 2. PubMed API请求频率限制

**问题描述**：PubMed API对请求频率有严格限制，无API密钥时每秒最多3个请求，有密钥时每秒最多10个请求。

**影响范围**：
- 论文导入脚本执行速度慢
- 大规模爬取论文时容易触发限制
- 可能导致导入任务中断

**当前状态**：
- 脚本中已实现请求延迟控制
- 支持配置 `PUBMED_API_KEY` 环境变量
- 实现了错误重试机制

**后续改进建议**：
- 配置PubMed API密钥以提高请求频率
- 实现更智能的请求队列管理
- 添加请求缓存避免重复查询
- 考虑使用其他学术论文API（如CrossRef、Europe PMC）

**相关文件**：
- `scripts/fetch-pubmed-papers.mjs` - 论文导入脚本
- `DEPLOYMENT_GUIDE.md` - PubMed API密钥配置说明

---

### 3. 论文图片加载失败

**问题描述**：某些论文可能没有可用的缩略图或期刊封面，导致论文卡片显示不完整。

**影响范围**：
- 论文展示页面的视觉效果
- 用户体验可能受到影响

**当前状态**：
- 已在 `PaperList.tsx` 中实现图片加载失败的降级方案
- 显示默认占位符或文字替代

**后续改进建议**：
- 实现自动生成论文缩略图（基于标题和作者）
- 从期刊官网爬取高质量图片
- 使用AI生成论文相关的配图
- 添加图片上传功能，允许管理员手动上传

**相关文件**：
- `client/src/components/PaperList.tsx` - 论文列表组件
- `drizzle/schema.ts` - research_papers表定义

### 4. 风险计算模型精度问题

**问题描述**：当前风险计算模型基于简单的加权平均，可能不够精准。需要使用真实飞行员数据进行训练和优化。

**影响范围**：
- 风险评估结果的准确性
- 个性化建议的质量
- 用户对系统的信任度

**当前状态**：
- 已实现基础的风险计算算法
- 包含16条以上的个性化建议规则
- 支持三维风险评分（职业特征、代谢指标、综合健康）

**后续改进建议**：
- 收集更多飞行员评估数据
- 使用机器学习模型（如随机森林、梯度提升）优化预测
- 与医学专家合作验证算法
- 定期更新模型参数
- 实现A/B测试对比不同算法效果

**相关文件**：
- `server/routers.ts` - 风险计算逻辑
- `shared/suggestionEngine.ts` - 建议生成引擎
- `shared/assessment.ts` - 评估数据类型定义

### 5. 数据库性能问题

**问题描述**：随着数据量增加，某些查询可能变慢。需要添加适当的索引和优化查询。

**影响范围**：
- 评估历史记录查询速度
- 研究论文搜索速度
- 系统整体响应时间

**当前状态**：
- 已使用Drizzle ORM进行基本查询优化
- 未添加专门的数据库索引

**后续改进建议**：
- 在 `assessments` 表的 `pilotId`, `userId`, `createdAt` 字段添加索引
- 在 `research_papers` 表的 `title`, `authors`, `publicationYear` 字段添加索引
- 实现查询结果缓存
- 定期分析慢查询日志
- 考虑使用读写分离或数据库分片

**相关文件**：
- `drizzle/schema.ts` - 数据库schema定义
- `server/db.ts` - 数据库查询函数

## 改进方向和优先级

### 第一阶段（1-2周）- 高优先级

这些改进应该立即进行，以确保系统的基本功能和稳定性。

#### 1. 执行PubMed论文导入

**任务**：运行论文导入脚本，建立初始论文库。

**步骤**：
1. 配置PubMed API密钥（可选但推荐）
2. 运行 `node scripts/fetch-pubmed-papers.mjs`
3. 验证论文是否正确导入到数据库
4. 测试论文展示页面功能

**预期结果**：系统中有100+篇真实的肾结石研究论文可供展示。

**相关文件**：
- `scripts/fetch-pubmed-papers.mjs`
- `client/src/pages/Papers.tsx`

---

#### 2. 配置定时论文更新任务

**任务**：实现每周自动爬取最新论文的定时任务。

**步骤**：
1. 安装node-cron库：`pnpm add node-cron`
2. 创建定时任务脚本 `server/jobs/updatePapers.ts`
3. 每周一凌晨2点执行论文更新
4. 添加日志记录和错误通知

**预期结果**：系统自动保持论文数据最新。

**相关文件**：
- `server/routers.ts` - 添加定时任务管理接口
- `scripts/fetch-pubmed-papers.mjs` - 论文导入逻辑

---

#### 3. 修复已知的UI/UX问题

**任务**：修复表单填写、报告显示等已知的UI问题。

**步骤**：
1. 收集用户反馈
2. 修复表单在移动设备上的显示问题
3. 优化报告页面的布局
4. 进行浏览器兼容性测试

**预期结果**：系统在各种设备和浏览器上都能正常使用。

---

#### 4. 完善错误处理和日志

**任务**：改进系统的错误处理和日志记录。

**步骤**：
1. 添加全局错误处理中间件
2. 实现结构化日志记录
3. 添加错误追踪和通知
4. 完善错误提示信息

**预期结果**：系统出现问题时能快速定位和解决。

---

### 第二阶段（2-4周）- 中优先级

这些改进可以在第一阶段完成后进行，以增强系统的功能和用户体验。

#### 1. 实现论文推荐功能

**任务**：根据用户评估结果推荐相关论文。

**步骤**：
1. 分析评估数据中的关键指标
2. 根据指标匹配相关论文
3. 实现推荐算法
4. 在评估结果页面展示推荐论文

**预期结果**：用户可以看到与其评估结果相关的研究论文。

---

#### 2. 集成LLM生成中文摘要

**任务**：使用内置LLM服务自动生成论文的中文摘要。

**步骤**：
1. 调用 `invokeLLM` 函数翻译论文摘要
2. 存储中文摘要到数据库
3. 在论文卡片中显示中文摘要
4. 实现定时批量处理

**预期结果**：用户可以阅读中文摘要，降低理解难度。

**相关文件**：
- `server/_core/llm.ts` - LLM集成
- `server/routers.ts` - 添加摘要生成接口

---

#### 3. 实现评估进度保存（草稿功能）

**任务**：允许用户保存评估进度，稍后继续填写。

**步骤**：
1. 添加draft状态到assessments表
2. 实现自动保存功能
3. 在首页显示草稿列表
4. 支持恢复和删除草稿

**预期结果**：用户可以分多次完成评估，提高完成率。

---

#### 4. 添加数据可视化增强

**任务**：添加更多交互式图表和数据可视化。

**步骤**：
1. 在评估结果页面添加更多图表
2. 实现数据对比功能
3. 添加趋势分析图表
4. 实现图表导出功能

**预期结果**：用户可以更直观地理解评估结果。

**相关文件**：
- `client/src/components/ResearchDashboard.tsx`
- `recharts` 库文档
