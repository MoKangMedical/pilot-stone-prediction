## 项目架构和技术栈

### 前端架构
- **框架**：React 19 + TypeScript
- **构建工具**：Vite 7
- **样式**：Tailwind CSS 4 + shadcn/ui组件库
- **状态管理**：tRPC + React Query
- **路由**：wouter
- **图表**：Recharts
- **动画**：Framer Motion

### 后端架构
- **运行时**：Node.js 22
- **框架**：Express 4
- **API框架**：tRPC 11
- **数据库ORM**：Drizzle 0.44
- **认证**：Manus OAuth + JWT
- **文件存储**：AWS S3

### 数据库
- **数据库系统**：MySQL 8.0 / TiDB 5.0
- **表数量**：8个主要表
- **数据类型**：用户、飞行员、评估、API配置、研究论文等

### 部署平台
- **平台**：Manus
- **托管方式**：全托管（无需手动配置服务器）
- **CI/CD**：自动构建和部署
- **域名**：自动生成 `*.manus.computer` 域名
- **HTTPS**：自动配置SSL证书

## 核心功能流程

### 风险评估流程

```
用户填写表单
    ↓
验证输入数据
    ↓
计算三维风险分数
    ↓
生成个性化建议
    ↓
保存评估记录到数据库
    ↓
生成PDF报告
    ↓
显示结果和建议
```

### 论文导入流程

```
运行导入脚本
    ↓
调用PubMed API搜索论文
    ↓
获取论文详情（标题、作者、摘要等）
    ↓
获取论文缩略图
    ↓
去重处理
    ↓
保存到数据库
    ↓
定时更新（每周）
```

### 用户认证流程

```
用户点击登录
    ↓
跳转到Manus OAuth门户
    ↓
用户输入凭证
    ↓
OAuth服务器验证
    ↓
返回授权码
    ↓
后端交换JWT Token
    ↓
设置Session Cookie
    ↓
用户登录成功
```

## 立即行动项（接管后的第一周）

### Day 1-2：环境熟悉

1. **阅读所有交接文档**
   - 理解项目架构和功能
   - 了解技术栈和工具

2. **本地环境设置**
   ```bash
   cd /home/ubuntu/pilot-stone-prediction
   pnpm install
   pnpm db:push
   pnpm run dev
   ```

3. **验证功能**
   - 访问 http://localhost:5173 测试前端
   - 测试评估功能
   - 测试论文展示功能

### Day 3-4：数据导入

1. **配置PubMed API密钥**（可选但推荐）
   - 访问 https://www.ncbi.nlm.nih.gov/account/
   - 生成API密钥
   - 在Manus Management UI配置

2. **执行论文导入**
   ```bash
   node scripts/fetch-pubmed-papers.mjs
   ```

3. **验证论文导入**
   - 检查数据库中的论文数量
   - 测试论文搜索和筛选功能
   - 验证论文图片加载

### Day 5：部署验证

1. **本地构建和测试**
   ```bash
   pnpm build
   pnpm start
   ```

2. **测试生产构建**
   - 验证所有功能正常
   - 检查性能
   - 检查错误日志

3. **部署到生产**
